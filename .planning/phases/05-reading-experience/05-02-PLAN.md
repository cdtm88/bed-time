    **File 2: `src/components/reading-view.tsx`** -- New file. `'use client'` component implementing the full reading experience.

    Structure and requirements:

    **State:**
    - `storyData: StoryData | null` (parsed from sessionStorage)
    - `error: boolean` (JSON parse failure)
    - `progress: number` (0-1 scroll progress)

    **StoryData interface** (define in this file):
    ```typescript
    interface StoryData {
      story: string
      name: string
      theme: string
    }
    ```

    **useEffect #1 -- SessionStorage read (on mount):**
    - Read `sessionStorage.getItem('nightlight-story')`
    - If null/missing: leave storyData as null (renders empty state)
    - If present: try `JSON.parse(raw)` and set storyData
    - If JSON.parse throws: set error=true (renders error state per D-04)

    **useEffect #2 -- Scroll tracking:**
    - Add `window.addEventListener('scroll', handleScroll, { passive: true })`
    - handleScroll: `const scrollTop = window.scrollY; const docHeight = document.documentElement.scrollHeight - window.innerHeight; setProgress(docHeight > 0 ? scrollTop / docHeight : 0)`
    - Cleanup: remove event listener on unmount

    **Rendering -- 3 states:**

    **State A: Error (error === true)**
    - Full viewport centered layout, bg-reading-surface
    - Heading: "Something went wrong" in display-lg (font-serif text-[2rem] font-normal leading-[1.3] tracking-[-0.02em]) text-reading-on-surface
    - Body: "Something went wrong loading the story. Please try generating a new one." in body-lg (font-serif text-[1rem] leading-[1.6]) text-reading-on-surface-muted
    - Button: gold pill linking to "/" with text "TRY AGAIN" -- same styling as New Story button below
    - Spacing: 8px gap between heading and body, 24px gap between body and button

    **State B: Empty (storyData === null and error === false)**
    - Full viewport centered layout (min-h-screen flex items-center justify-center), bg-reading-surface
    - Heading: "No story yet" in display-lg (font-serif text-[2rem] font-normal leading-[1.3] tracking-[-0.02em]) text-reading-on-surface
    - Body: "Head back to create a bedtime story." in body-lg (font-serif text-[1rem] leading-[1.6]) text-reading-on-surface-muted
    - Button: `<a href="/" aria-label="Create a new bedtime story">` styled as gold pill -- bg-secondary-container text-on-secondary-container font-sans text-[0.75rem] font-semibold uppercase tracking-[0.04em] h-[48px] px-xl rounded-full inline-flex items-center justify-center transition-all duration-300 ease-in-out hover:brightness-110
    - Button text: "CREATE A STORY"
    - Spacing: 8px gap between heading and body (mt-sm), 24px gap between body and button (mt-lg)

    **State C: Story loaded (storyData !== null)**

    Outer wrapper: `<div className="min-h-screen bg-reading-surface">` -- this overrides the body's bg-surface for the full page.

    Scroll progress bar (per D-05):
    ```html
    <div
      aria-hidden="true"
      className="fixed top-0 left-0 h-[3px] z-40 transition-opacity duration-300 ease-in-out"
      style={{
        width: `${progress * 100}%`,
        backgroundColor: 'var(--color-reading-on-surface-muted)',
        opacity: progress > 0 ? 0.4 : 0,
      }}
    />
    ```

    Fade-in animation (inline `<style>` block, following loading-overlay.tsx pattern):
    ```html
    <style>{`
      @keyframes fadeIn {
        from { opacity: 0; }
        to { opacity: 1; }
      }
    `}</style>
    ```

    Content container: `<div style={{ animation: 'fadeIn 600ms ease-in-out' }} className="mx-auto max-w-[640px] px-lg pt-2xl pb-3xl">`

    Layout breakdown inside content container:

    1. **Story title** (per D-03): `<article role="article"><h1>` with text `assembleTitle(storyData.name, storyData.theme)` which produces "{Name}'s {Theme} Story".
       - Classes: `font-serif text-[2rem] font-normal leading-[1.3] tracking-[-0.02em] text-reading-on-surface-muted`
       - Margin-bottom: `mb-xl` (32px)

    2. **Story body**: Map paragraphs from `storyData.story.split('\n\n').filter(p => p.trim())` into `<p>` elements.
       - Each `<p>`: `font-serif text-[1.25rem] font-normal leading-[1.8] tracking-[0.01em] text-reading-on-surface mb-md`
       - Last paragraph: no margin-bottom (use `last:mb-0` or conditional)
       - Add `style={{ textRendering: 'optimizeLegibility', WebkitFontSmoothing: 'antialiased' }}` on the article or a wrapper div

    3. **End-of-story section**: After the `</article>`, 48px top margin (`mt-2xl`).
       - "New Story" button: `<a href="/">` styled as gold pill (same as empty state button).
         - Classes: `bg-secondary-container text-on-secondary-container font-sans text-[0.75rem] font-semibold uppercase tracking-[0.04em] h-[48px] px-xl rounded-full inline-flex items-center justify-center transition-all duration-300 ease-in-out hover:brightness-110`
         - Text: "NEW STORY"
         - Wrap in a centered div: `<div className="flex flex-col items-center gap-md">`
       - Back link (per D-06): `<a href="/" className="font-sans text-[0.75rem] text-reading-on-surface-muted hover:text-reading-on-surface transition-colors duration-300">` with text content like the unicode left arrow followed by " Home"
       - Focus ring on both: `focus-visible:outline-2 focus-visible:outline-reading-on-surface-muted focus-visible:outline-offset-2`

    Close the `</article>` after story paragraphs. The end-of-story section is outside the article element.

    **Accessibility per UI-SPEC:**
    - Story content wrapped in `<article role="article">`
    - Title is `<h1>`
    - Story body uses `<p>` elements (NOT whitespace-pre-wrap on single block)
    - Scroll progress: `aria-hidden="true"`
    - Empty state button: `aria-label="Create a new bedtime story"`
    - Focus rings on all interactive elements

    **Do NOT:**
    - Add any floating toolbar (rejected per D-05)
    - Add screen wake lock (deferred, out of scope)
    - Set maximumScale or userScalable on viewport (accessibility anti-pattern per RESEARCH.md)
    - Use whitespace-pre-wrap on a single text block (anti-pattern per RESEARCH.md)
