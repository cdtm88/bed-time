## Typography

| Role | Token | Font | Size | Weight | Line Height | Letter Spacing |
|------|-------|------|------|--------|-------------|----------------|
| Story title / Empty state heading | `display-lg` | Noto Serif | 2rem (32px) | 400 (Regular) | 1.3 | -0.02em |
| Story body | `body-reading` | Noto Serif | 1.25rem (20px) | 400 (Regular) | 1.8 | 0.01em |
| Empty state body | `body-lg` | Noto Serif | 1rem (16px) | 400 (Regular) | 1.6 | normal |
| UI control label | `label-md` | Plus Jakarta Sans | 0.75rem (12px) | 600 (Semi-Bold) | 1.3 | 0.04em |

**Weight palette:** 400 (Regular) and 600 (Semi-Bold) only. Consistent with Phase 4.

**Reading-optimized body text:** Story body uses 20px at 1.8 line-height -- larger than the form's 16px body. This is intentional: parents read aloud from arm's length in dim light. The 1.8 line-height provides generous inter-line spacing that prevents losing your place mid-sentence. The 0.01em letter spacing opens up readability at this size without feeling tracked-out.

Usage in this phase:
- **Story title** (child's name + theme context): `display-lg` at 32px -- smaller than the full 56px display to leave room for the story
- **Story paragraphs**: `body-reading` (20px, 1.8 line-height)
- **"New Story" button text**: `label-md`, uppercase
- **Empty state heading**: `display-lg` (32px) -- shares token with story title for consistency
- **Empty state body**: `body-lg`

## Color

This phase introduces a **dark reading mode** palette. The form phase uses warm cream surfaces; the reading phase inverts to warm dark surfaces for dim-room comfort. These are new tokens added alongside the existing light tokens.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `reading-surface` | #1a1a2e | Deep warm navy -- main page background |
| Secondary (30%) | `reading-surface-elevated` | #232340 | Floating toolbar background, story title area |
| Text (primary) | `reading-on-surface` | #e8e0d0 | Story body text -- warm off-white, not harsh pure white |
| Text (muted) | `reading-on-surface-muted` | #a89f8f | Story title, metadata, control labels |
| Accent (10%) | `secondary-container` | #fcd400 | "New Story" button fill (celebratory gold per DESIGN.md) |
| Accent text | `on-secondary-container` | #1b1c17 | Text on gold button |
| Destructive | `destructive` | #b3261e | Error text (reused from Phase 4) |

**Dark mode rationale:** READ-01 requires "warm/dark background colors, readable in a dim room." A dark navy (#1a1a2e) with warm undertones avoids the cold, clinical feel of pure black or neutral gray. The warm off-white text (#e8e0d0) reduces blue-light emission compared to pure white (#ffffff), which matters for a pre-sleep context.

**Contrast verification:** `reading-on-surface` (#e8e0d0) on `reading-surface` (#1a1a2e) provides a contrast ratio of approximately 11.5:1, exceeding WCAG AAA (7:1) for the 20px body text.

Accent reserved for:
- "New Story" button (gold fill -- celebratory moment of completing a story per DESIGN.md)
- No other accent usage in this phase -- the reading experience is intentionally monochromatic to minimize distraction

## Visual Hierarchy and Focal Point

**Primary visual anchor:** The story text itself. Unlike Phase 4 where the CTA button was the focal point, this phase centers the narrative content as the dominant element.

**Visual hierarchy:**
1. Story body text -- largest text block, highest contrast, occupies 80%+ of viewport
2. Story title -- muted gold tone, smaller than body's visual weight, orients the reader
3. "New Story" button -- gold accent, bottom of page, visible only after scrolling through the story
4. Floating toolbar (if visible) -- semi-transparent, intentionally subdued

**Reading flow:** Title establishes context (child's name, theme), then body text fills the viewport as a continuous, scrollable narrative. The "New Story" button appears at the end as the natural next action after finishing the story.

## Component Inventory

### 1. Reading Container (Full Viewport)
- Position: min-height 100vh, no scrollbar styling (native scroll)
- Background: `reading-surface` (#1a1a2e)
- No navigation chrome, no header, no footer visible during reading
- Padding: horizontal 24px (mobile), max-width 640px centered (optimal line length ~60-70 characters at 20px)
- Transition: page entry uses 600ms fade-in from black (ease-in-out) to gently introduce the reading environment

### 2. Story Title Block
- Position: top of reading container, below 48px top padding
- Text: `display-lg` (32px), `reading-on-surface-muted` (#a89f8f)
- Content: "{Child's Name}'s {Theme} Adventure" or similar contextual title
- Margin-bottom: 32px below title before first paragraph
- No border, no background -- text only, floating on the dark surface

### 3. Story Body
- Text: `body-reading` (Noto Serif, 20px, 400 weight, 1.8 line-height, 0.01em letter spacing)
- Color: `reading-on-surface` (#e8e0d0)
- Paragraph spacing: 16px margin-bottom between paragraphs
- Text rendering: `text-rendering: optimizeLegibility`, `font-smoothing: antialiased`
- Word spacing: normal (no adjustment needed at this size)
- Max-width: 640px (enforced by container, not on text element)
- Paragraphs are split by double newline from the API response and rendered as separate `<p>` elements

### 4. End-of-Story Section
- Position: after last paragraph, with 48px top margin
- Separator: none (no horizontal rules per DESIGN.md)
- Visual break: 48px of empty space signals story completion
- "New Story" button: centered, pill shape (`border-radius: 9999px`), gold fill (`secondary-container` #fcd400), dark text (`on-secondary-container` #1b1c17)
- Button text: `label-md`, uppercase -- "NEW STORY"
- Button height: 48px, horizontal padding 32px
- Button action: navigates to `/` (home/form page)
- Below button: 64px bottom padding
- Transition: button hover brightness +10%, 300ms ease-in-out

### 5. Empty State (No Story Found)
- Triggers when: sessionStorage has no `nightlight-story` key
- Background: `reading-surface` (#1a1a2e) -- consistent with reading mode
- Layout: centered vertically and horizontally
- Heading: `display-lg` (32px), `reading-on-surface` (#e8e0d0) -- "No story yet"
- Body: `body-lg` (16px), `reading-on-surface-muted` (#a89f8f) -- "Head back to create a bedtime story."
- Action button: same gold pill as "New Story" -- "CREATE A STORY"
- Spacing: 8px gap between heading and body, 24px gap between body and button

### 6. Scroll Progress Indicator (Subtle)
- Position: fixed top of viewport, full width, 3px height
- Color: `reading-on-surface-muted` (#a89f8f) at 40% opacity
- Fill: left-to-right, proportional to scroll position
- Disappears (opacity 0) when at scroll top (0%) -- only appears once reading begins
- Transition: width changes are smooth (no transition needed -- follows scroll directly), opacity 300ms ease-in-out
- Z-index: 40

## Interaction States

### Page Entry Flow
1. Parent arrives at `/story` after generation completes (via `window.location.href` from Phase 4)
2. Page reads `nightlight-story` from sessionStorage
3. If story exists: render reading container with 600ms fade-in
4. If no story: render empty state immediately (no fade)

### Scroll Behavior
- Native browser scroll, no custom scroll hijacking
- Scroll progress indicator tracks position (0% at top, 100% at bottom)
- No scroll snap -- continuous smooth scroll

### Navigation
- "New Story" button at end of story navigates to `/`
- Browser back button returns to `/` (standard behavior, no interception)
- No other navigation controls -- intentional chrome-free design per READ-01

### Viewport Meta
- Story page should set `<meta name="theme-color" content="#1a1a2e">` to color the browser chrome on mobile, reinforcing the dark reading environment
- Future (v2): Wake Lock API prevents screen sleep (READ-03, out of scope for v1)

### Accessibility
- Story content wrapped in `<article>` with `role="article"`
- Story title uses `<h1>` element
- Story paragraphs use `<p>` elements (not `whitespace-pre-wrap` on a single block)
- "New Story" button is a standard `<a>` or `<button>` with clear focus ring (2px `reading-on-surface-muted` outline, 2px offset)
- Scroll progress indicator has `aria-hidden="true"` (decorative)
- Empty state action button has `aria-label="Create a new bedtime story"`
- Color contrast exceeds WCAG AAA at all text sizes used

## Layout

### Mobile-First (320px-768px)
- Reading container: full width, horizontal padding 24px
- Story text max-width: none (constrained by container padding)
- Effective line length: ~55-65 characters at 20px (optimal for reading)
- Title: left-aligned
- Story text: left-aligned (not justified -- avoids uneven word spacing)
- "New Story" button: centered

### Tablet+ (769px+)
- Reading container: max-width 640px, centered with `margin: 0 auto`
- Horizontal padding: 0 (max-width provides the constraint)
- No other layout changes -- reading is inherently a narrow-column experience
