Build the parent-facing input form at / with all interactive controls, loading overlay, and story handoff.

Purpose: This is the core user interface -- the first thing a parent sees when they visit Nightlight Tales. The form collects the child's name, age, theme, and story duration, validates in real-time, submits to /api/generate, shows a calming loading overlay during generation (~30s), and navigates to /story on success. No login, no friction (INFRA-01).

Output: Six UI components (StoryForm, NameInput, AgeStepper, DurationToggle, ThemeGrid, LoadingOverlay) and the updated page.tsx landing page.

    **1. Create src/components/story-form.tsx** (the main form orchestrator)

    This is a `'use client'` component that:
    - Manages all form state with `useState`:
      - `name: string` (default: `""`)
      - `age: number` (default: `5` -- per UI-SPEC field defaults)
      - `duration: number` (default: `10` -- per UI-SPEC field defaults)
      - `theme: string | null` (default: `null` -- per UI-SPEC, parent must choose)
      - `isLoading: boolean` (default: `false`)
      - `error: string | null` (default: `null`)
    - Derives `canSubmit`: `name.trim().length > 0 && /^[a-zA-Z\s]{1,30}$/.test(name.trim()) && theme !== null`
    - Implements `handleSubmit` async function:
      1. Guard: `if (isLoading || !canSubmit) return`
      2. `setIsLoading(true)` and `setError(null)`
      3. `try`:
         - `const res = await fetch('/api/generate', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ name: name.trim(), age, theme, duration }) })`
         - If `res.status === 429`: `setIsLoading(false)`, `setError("You've created a few stories recently. Try again in a bit.")`, `return`
         - If `!res.ok`: `setIsLoading(false)`, `setError("Something went wrong creating the story. Please try again.")`, `return`
         - `const storyText = await res.text()`
         - `sessionStorage.setItem('nightlight-story', storyText)`
         - `sessionStorage.setItem('nightlight-story-name', name.trim())` (for Phase 5 to use)
         - `window.location.href = '/story'`
      4. `catch`: `setIsLoading(false)`, `setError("Something went wrong creating the story. Please try again.")`
    - Renders conditionally:
      - If `isLoading`: render `<LoadingOverlay name={name.trim()} />` (form unmounted per D-11)
      - Else: render the form

    Form layout (per D-01 single-scroll, D-02 field order, UI-SPEC Layout):
    ```
    <main className="min-h-screen bg-surface">

        {/* Title */}

          Nightlight Tales
        </h1>
        <p className="text-center font-serif text-[1rem] leading-relaxed text-on-surface/70 mt-sm">
          A bedtime story, just for them.
        </p>

        {/* Form fields with spacing */}
        <div className="mt-2xl flex flex-col gap-md">
          <NameInput value={name} onChange={setName} />
          <AgeStepper value={age} onChange={setAge} />
          <DurationToggle value={duration} onChange={setDuration} />
        </div>

        {/* Theme grid with larger gap above */}
        <div className="mt-2xl">
          <ThemeGrid value={theme} onChange={setTheme} />
        </div>

        {/* Generate button */}
        <button
          onClick={handleSubmit}
          disabled={!canSubmit}
          aria-disabled={!canSubmit}
          className="mt-xl w-full h-[52px] rounded-full bg-gradient-to-br from-primary to-primary-container font-sans text-[0.75rem] font-semibold uppercase tracking-[0.04em] text-white transition-all duration-300 ease-in-out hover:brightness-110 disabled:opacity-40 disabled:cursor-not-allowed"

          <p role="alert" className="mt-sm font-serif text-[1rem] leading-relaxed text-destructive">

          </p>
        )}
      </div>
    </main>
    ```
